# 소송서면 증거번호 자동 정렬기 (exhibit_renumber.py)

서면(`.hwpx`) 파일을 수정하다 보면 **증거**의 번호 순서가 바뀌는 경우나, 증거가 삽입되고 삭제되기도 합니다.
이 도구는 본문에 등장하는 순서대로 자동으로 번호를 재정렬해주고 서면 마무리 부분(입증방법, 소명방법, 참고자료 등)에 정리합니다.
또한 새로운 증거가 삽입되었는데 서면 마무리 부분에만 있고 본문에 없을 경우 해당 내용을 경고합니다.

**문서 유형은 자동으로 판별됩니다.** 스크립트 하나로 소장(갑 제N호증), 답변서(을 제N호증), 변호인의견서(참고자료 N) 모두 처리합니다.
`.hwpx` 파일과 증거 자료를 같은 폴더에 넣고 `exhibit_renumber.py`를 더블클릭하면 됩니다.

---

## 어떤 일을 해주나요?

- 본문에서 **증거**가 처음 등장하는 순서대로 1번부터 다시 번호를 매김
- 같은 증거가 본문에 여러 번 나오면 → 번호는 그대로 유지 (고정)
- **동일 이름 증거 자동 병합**: 같은 이름의 증거가 여러 번호로 있으면 첫 번째 번호로 통합 (예: 갑 제3호증, 갑 제6호증이 모두 "공급계약서"면 → 갑 제3호증으로 병합)
- 문서 끝의 **증거목록**(ex: 입증방법, 소명방법, 참고자료 등)도 새 순서대로 자동 재생성
- **인용 라인만 증거 이름으로 인식**: 본문 중 "갑 제8호증 녹취록의 내용을 보면..."처럼 문장 속에 등장하는 증거 번호는 이름으로 수집하지 않고, `- 갑 제8호증 녹취록` 형태의 인용 라인에서만 증거 이름을 추출
- 본문에 없는 증거가 목록에만 있으면 **경고 메시지**로 알려줌
- **이름 없는 자리표시자 무시**: 입증방법에 번호만 있고 이름이 없는 항목은 자리표시자로 판단하여 무시 (팬텀 증거 방지)
- 같은 폴더의 **증거 파일 이름**에 증거 번호를 자동 부여하고, 서면 기준으로 파일 이름을 통일
- 증거 파일을 **문서 유형별 하위 폴더**로 자동 정리
- **번호 없는 증거**(갑 제호증, 을 제호증)를 자동 인식하여 임시 번호 부여 후 정렬
- **소장**(갑 제N호증), **답변서**(을 제N호증), **변호인의견서/고발장**(참고자료 N) 세 가지 형식 자동 판별 지원

### 작성 팁

서면을 작성할 때 증거 번호를 처음부터 확정하지 말고, 아래 예시처럼 **번호 없이** 작성해두면 나중에 이 도구로 자동 정렬할 때 훨씬 편합니다.

```
- 갑 제호증 매매계약서        ← 번호 없이 작성
- 갑 제호증 계좌이체내역
```

### 예시

갑 제N호증, 을 제N호증, 참고자료 N 모두 동일한 방식으로 처리됩니다.
본문 등장 순서대로 번호를 1번부터 다시 매기고, 마무리 목록도 자동 재생성합니다.

```
[수정 전]
  피고는 원고에게 매매대금을 지급하였고,
  이에 대한 증빙은 아래와 같습니다.
  - 갑 제3호증 매매계약서

  원고의 계좌로 입금된 내역은 다음과 같습니다.
  - 갑 제1호증 계좌이체내역

  당사자 간 대화 내용에서도 이를 확인할 수 있습니다.
  - 갑 제5호증 카카오톡 대화

  입증방법
  1. 갑 제3호증  매매계약서
  2. 갑 제1호증  계좌이체내역
  3. 갑 제5호증  카카오톡 대화

[수정 후]
  피고는 원고에게 매매대금을 지급하였고,
  이에 대한 증빙은 아래와 같습니다.
  - 갑 제1호증 매매계약서

  원고의 계좌로 입금된 내역은 다음과 같습니다.
  - 갑 제2호증 계좌이체내역

  당사자 간 대화 내용에서도 이를 확인할 수 있습니다.
  - 갑 제3호증 카카오톡 대화

  입증방법
  1. 갑 제1호증 매매계약서
  2. 갑 제2호증 계좌이체내역
  3. 갑 제3호증 카카오톡 대화
```

### 참고자료 형식 (고발장, 자수서, 변호인의견서 등)

참고자료 모드에서는 입증방법과 다른 형식을 사용합니다.

```
[입증방법 형식 — 소장, 준비서면]
  1. 갑 제1호증 매매계약서
  2. 갑 제2호증 계좌이체내역

[참고자료 형식 — 고발장, 자수서, 변호인의견서]
  참고자료 1. 이지우 매니저 대화내역
  참고자료 2. 빗썸어플 충전내역
```

### 증거 파일 자동 이름 변경 및 폴더 정리

같은 폴더에 증거 자료 파일을 함께 넣어두면, 서면의 증거 번호에 맞추어 파일 이름도 자동으로 변경되고, **문서 유형에 따른 하위 폴더**로 정리됩니다.

```
[실행 전 폴더]
  (남양주)박대영(원).분양대금반환.소장.hwpx
  매매계약서.pdf
  계좌이체내역.xlsx
  카카오톡 대화.jpg

[실행 후 폴더 — 소장/준비서면]
  (남양주)박대영(원).분양대금반환.소장.hwpx               (원본 유지)
  (남양주)박대영(원).분양대금반환.소장_숫자정렬완.hwpx      (정렬 완료본)
  (남양주)박대영(원).분양대금반환.소장_입증방법/            ← 자동 생성 폴더
    갑 제1호증 매매계약서.pdf
    갑 제2호증 계좌이체내역.xlsx
    갑 제3호증 카카오톡 대화.jpg

[실행 후 폴더 — 고발장/자수서]
  여수빈(고발).보이스피싱.고발장.hwpx                     (원본 유지)
  여수빈(고발).보이스피싱.고발장_숫자정렬완.hwpx            (정렬 완료본)
  여수빈(고발).보이스피싱.고발장_참고자료/                  ← 자동 생성 폴더
    참고자료 1. 이지우 매니저 대화내역.pdf
    참고자료 2. 빗썸어플 충전내역.png
```

- 서면의 입증방법/참고자료 목록에 있는 증거 이름과 파일 이름을 대조하여 자동 매칭
- **서면 기준 파일 이름 통일**: 파일 이름과 서면의 증거 이름이 띄어쓰기, 날짜 표기(04. → 4.) 등에서 차이가 있어도 자동 매칭 후 서면 기준으로 통일
- **번호 뒤 숫자 보호**: "카카오톡 내역1"과 "카카오톡 내역2"는 서로 다른 증거로 취급 (숫자 접미사 구분)
- 이미 번호가 붙어 있는 파일은 기존 접두사를 벗기고 새 번호로 교체
- 번호가 서로 바뀌는 경우(1↔2)에도 충돌 없이 안전하게 이름 변경 (2-pass 방식)
- 하위 폴더명은 문서 유형에 따라 자동 결정: 소장/준비서면 → `_입증방법`, 고발장/자수서 등 → `_참고자료`
- 재실행 시 기존 하위 폴더(이전 버전의 `_자료` 폴더 포함)에서도 파일을 탐색
- 일치하는 파일이 없으면 건너뜀
- `.hwpx`, `.py` 파일은 자동 제외

## 사용 방법

### 방법 1. 더블클릭 (가장 간단)

1. 다운받은 폴더 안에 `.hwpx` 파일과 증거 자료 파일을 함께 넣어주세요.
2. `exhibit_renumber.py`를 **더블클릭**하면 폴더 안의 모든 `.hwpx` 파일을 자동으로 처리합니다.
3. 같은 폴더에 `_숫자정렬완.hwpx` 파일이 새로 생성되고, 증거 파일은 하위 폴더로 자동 정리됩니다.

### 방법 2. PowerShell (미리보기·개별 파일 처리)

폴더 안에서 **빈 공간 우클릭 → "터미널에서 열기"**

**미리보기** (실제 파일은 안 바뀜):
```powershell
$env:PYTHONIOENCODING="utf-8"
python exhibit_renumber.py --preview "서면파일.hwpx"
```

**실제 변환**:
```powershell
python exhibit_renumber.py "서면파일.hwpx"
```

## 자주 묻는 질문

**Q. 한글이 깨져 보여요**
A. PowerShell에서 `$env:PYTHONIOENCODING="utf-8"` 명령을 먼저 입력하세요.

**Q. "찾을 수 없습니다" 에러가 나요**
A. 폴더 경로에 대괄호(`[]`)가 있으면 `cd` 대신 `Set-Location -LiteralPath '경로'`를 써주세요.

**Q. 입증방법 목록의 1번, 2번이 안 보여요**
A. 한글 파일에서 자동번호 매기기 스타일이라면 텍스트에는 번호가 없어도 화면에서는 자동으로 보입니다. 정상입니다.

**Q. `.hwp` 파일도 되나요?**
A. 아니요. `.hwpx` (한글 2010 이후 형식)만 지원합니다. 한글에서 **다른 이름으로 저장 → 파일 형식: HWPX**로 변환 후 사용해주세요.

**Q. 증거 파일 이름이 안 바뀌어요**
A. 파일 이름과 서면의 증거 이름이 유사하면 자동 매칭됩니다. 띄어쓰기, 날짜 표기(04. → 4.) 차이는 자동 보정됩니다. 다만 완전히 다른 이름이면 매칭되지 않으니, 서면의 증거 이름과 비슷하게 파일 이름을 맞춰주세요.

**Q. 같은 이름의 증거가 여러 개 있으면 어떻게 되나요?**
A. 동일한 이름의 증거는 첫 번째 등장한 번호로 자동 병합됩니다. 예를 들어 갑 제3호증과 갑 제6호증이 모두 "공급계약서"이면, 갑 제6호증이 갑 제3호증으로 통합됩니다.

## 작동 원리

1. HWPX(ZIP) 안의 `Contents/section0.xml` 파싱
2. 문서 유형 자동 판별 (갑호증 / 을호증 / 참고자료)
3. 번호 없는 증거(갑 제호증, 을 제호증) 자동 인식 → 임시 번호 부여
4. **증거목록** 섹션 자동 탐지 (오탐 방지를 위해 단락 단독 키워드 + 다음 줄 패턴 두 조건 확인)
5. 마무리 섹션 **이전** 본문에서 증거 최초 등장 순서 수집 (인용 라인만 이름으로 인식)
6. 동일 이름 증거 자동 병합 (첫 번째 등장 번호로 통합)
7. 이름 없는 자리표시자 항목 필터링
8. 새 번호 매핑 생성 후 본문 전체 치환
9. 마무리 목록을 새 순서대로 재생성 (갑/을호증: `1. 갑 제1호증 이름`, 참고자료: `참고자료 1. 이름`)
10. 같은 폴더의 증거 파일을 서면 기준 이름으로 통일 후 문서 유형별 하위 폴더로 이동 (2-pass 충돌 방지)
11. 새 HWPX 파일로 저장
