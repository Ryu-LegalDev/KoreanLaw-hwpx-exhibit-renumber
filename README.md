# 소송서면 증거번호 자동 정렬기 (exhibit_renumber.py)

서면(`.hwpx`) 파일을 수정하다 보면 **증거**의 번호 순서가 바뀌는 경우나, 증거가 삽입되고 삭제되기도 합니다.
이 도구는 본문에 등장하는 순서대로 자동으로 번호를 재정렬해주고 서면 마무리 부분(입증방법, 소명방법, 참고자료 등)에 정리합니다.
또한 새로운 증거가 삽입되었는데 서면 마무리 부분에만 있고 본문에 없을 경우 해당 내용을 경고하는 파이썬 스크립트입니다.

**문서 유형은 자동으로 판별됩니다.** 스크립트 하나로 소장(갑 제N호증)과 변호인의견서(참고자료 N) 모두 처리합니다.

---

## 어떤 일을 해주나요?

- 본문에서 **증거**가 처음 등장하는 순서대로 1번부터 다시 번호를 매김
- 같은 증거가 본문에 여러 번 나오면 → 번호는 그대로 유지 (고정)
- 문서 끝의 **증거목록**(ex: 입증방법, 소명방법, 참고자료 등)도 새 순서대로 자동 재생성
- 본문에 없는 증거가 목록에만 있으면 **경고 메시지**로 알려줌
- **소장**(갑 제N호증), **답변서**(을 제N호증), **변호인의견서**(참고자료 N) 세 가지 형식 자동 판별 지원

### 작성 팁

서면을 작성할 때 증거 번호를 처음부터 확정하지 말고, 아래 예시처럼 **번호 없이** 작성해두면 나중에 이 도구로 자동 정렬할 때 훨씬 편합니다.

```
- 갑 제호증 매매계약서        ← 번호 없이 작성
- 갑 제호증 계좌이체내역
```

### 예시

갑 제N호증, 을 제N호증, 참고자료 N 모두 동일한 방식으로 처리됩니다.
본문 등장 순서대로 번호를 1번부터 다시 매기고, 마무리 목록도 자동 재생성합니다.

```
[수정 전]
  피고는 원고에게 매매대금을 지급하였고,
  이에 대한 증빙은 아래와 같습니다.
  - 갑 제3호증 매매계약서

  원고의 계좌로 입금된 내역은 다음과 같습니다.
  - 갑 제1호증 계좌이체내역

  당사자 간 대화 내용에서도 이를 확인할 수 있습니다.
  - 갑 제5호증 카카오톡 대화

  입증방법
  1. 갑 제3호증  매매계약서
  2. 갑 제1호증  계좌이체내역
  3. 갑 제5호증  카카오톡 대화

[수정 후]
  피고는 원고에게 매매대금을 지급하였고,
  이에 대한 증빙은 아래와 같습니다.
  - 갑 제1호증 매매계약서

  원고의 계좌로 입금된 내역은 다음과 같습니다.
  - 갑 제2호증 계좌이체내역

  당사자 간 대화 내용에서도 이를 확인할 수 있습니다.
  - 갑 제3호증 카카오톡 대화

  입증방법
  1. 갑 제1호증  매매계약서
  2. 갑 제2호증  계좌이체내역
  3. 갑 제3호증  카카오톡 대화
```

## 사용 전 준비

### 1. Python 설치
- https://www.python.org/downloads/ 접속
- **Download Python 3.x.x** 클릭 → 설치
- 설치 화면에서 **"Add Python to PATH"** 반드시 체크 후 Install

### 2. 코드 받기
이 페이지 상단의 초록색 **`<> Code`** 버튼 → **Download ZIP** → 압축 풀기

(Git을 설치했다면: `git clone https://github.com/사용자명/저장소명`)

## 사용 방법

### 단계 1. 서면 파일을 코드 폴더에 넣기

다운받은 폴더 안에 정렬하고 싶은 `.hwpx` 파일을 복사해 넣어주세요.

### 단계 2. PowerShell 열기

폴더 안에서 **빈 공간 우클릭 → "터미널에서 열기"** (Windows 11)
또는 Shift + 우클릭 → "PowerShell 창 여기서 열기" (Windows 10)

### 단계 3. 미리보기 (실제 파일은 안 바뀜)

```powershell
$env:PYTHONIOENCODING="utf-8"
python exhibit_renumber.py --preview "서면파일.hwpx"
```

문서 유형(갑호증 / 을호증 / 참고자료)은 자동으로 판별됩니다. 번호가 어떻게 바뀔지 미리 확인할 수 있어요.

### 단계 4. 실제 변환

```powershell
python exhibit_renumber.py "서면파일.hwpx"
```

같은 폴더에 **`서면파일_숫자정렬완.hwpx`** 가 새로 생성됩니다. 원본은 그대로 유지돼요.

## 자주 묻는 질문

**Q. 한글이 깨져 보여요**
A. PowerShell에서 `$env:PYTHONIOENCODING="utf-8"` 명령을 먼저 입력하세요.

**Q. "찾을 수 없습니다" 에러가 나요**
A. 폴더 경로에 대괄호(`[]`)가 있으면 `cd` 대신 `Set-Location -LiteralPath '경로'`를 써주세요.

**Q. 입증방법 목록의 1번, 2번이 안 보여요**
A. 한글 파일에서 자동번호 매기기 스타일이라면 텍스트에는 번호가 없어도 화면에서는 자동으로 보입니다. 정상입니다.

**Q. `.hwp` 파일도 되나요?**
A. 아니요. `.hwpx` (한글 2010 이후 형식)만 지원합니다. 한글에서 **다른 이름으로 저장 → 파일 형식: HWPX**로 변환 후 사용해주세요.

## 작동 원리

1. HWPX(ZIP) 안의 `Contents/section0.xml` 파싱
2. 문서 유형 자동 판별 (갑호증 / 을호증 / 참고자료)
3. **증거목록** 섹션 자동 탐지 (오탐 방지를 위해 단락 단독 키워드 + 다음 줄 패턴 두 조건 확인)
4. 마무리 섹션 **이전** 본문에서 증거 최초 등장 순서 수집
5. 새 번호 매핑 생성 후 본문 전체 치환
6. 마무리 목록을 한글 자동번호 스타일로 통일 재생성
7. 새 HWPX 파일로 저장
